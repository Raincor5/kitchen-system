Overview

The Recipe Upscaler API provides endpoints for intelligent recipe scaling, unit conversion, and nutritional analysis. The API uses advanced algorithms to accurately scale ingredients while preserving the recipe's integrity.

Base URL

https://api.recipe-upscaler.example.com/v1

Response Format

All responses are returned in JSON format with the following structure:

{
  "status": "success",
  "data": { ... },
  "message": "Operation successful"
}

HTTP Status Codes

  • 200 OK - The request was successful
  • 201 Created - The resource was successfully created
  • 400 Bad Request - The request was invalid
  • 401 Unauthorized - Authentication failed
  • 403 Forbidden - The client does not have permission
  • 404 Not Found - The resource was not found
  • 500 Internal Server Error - An error occurred on the server

Authentication

The API uses API keys for authentication. Include your API key in the header of each request:

Authorization: Bearer YOUR_API_KEY
Contact the administrator to get an API key for your application.

Error Handling

When an error occurs, the API returns a JSON response with the following structure:

{
  "status": "error",
  "error": {
    "code": "validation_error",
    "message": "Invalid request parameters",
    "details": {
      "target_servings": "Must be a positive number"
    }
  }
}

Common Error Codes

  • validation_error - Invalid request parameters
  • authentication_error - Authentication failed
  • permission_error - The client does not have permission
  • not_found - The resource was not found
  • conversion_error - Unable to convert between the specified units
  • scaling_error - Unable to scale the recipe
  • server_error - An error occurred on the server
